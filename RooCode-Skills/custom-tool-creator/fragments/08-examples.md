# 08-examples: Ready-to-Use Tool Templates

Этот раздел содержит готовые шаблоны инструментов для частых задач.

## Клиент API

**Файл:** [`api-client.js`](../assets/examples/api-client.js)  
REST клиент с повторными попытками, таймаутами и обработкой ошибок. Идеален для интеграции с внешними API. Включает валидацию URL для предотвращения SSRF и белый список хостов.

## Операции с файлами

**Файл:** [`file-ops.js`](../assets/examples/file-ops.js)  
Безопасные операции чтения и записи файлов в рамках рабочей области. Ограничивает размер файлов, проверяет доступ и автоматически создаёт родительские директории.

## Пакетные операции

**Файл:** [`batch-ops.js`](../assets/examples/batch-ops.js)  
Массовое переименование файлов с использованием регулярных выражений. Поддерживает dry-run режим для предпросмотра изменений и проверяет, что директория находится в рабочей области.

## Операции с Git

**Файл:** [`git-ops.js`](../assets/examples/git-ops.js)  
Инструменты для работы с Git с белым списком разрешённых команд. Включает статус репозитория и создание коммитов с опциональным переключением ветки и исправлением предыдущих коммитов.

## Уведомления Slack

**Файл:** [`slack-notify.js`](../assets/examples/slack-notify.js)  
Отправка уведомлений в Slack через вебхук. Требует переменную окружения `SLACK_WEBHOOK_URL`. Поддерживает переопределение канала и выбор эмодзи.

## Преобразование данных

**Файл:** [`transform.js`](../assets/examples/transform.js)  
Конвертация данных между JSON, CSV и XML форматами. Использует библиотеки `json2csv` и `xml2js`. Поддерживает настройки CSV (разделитель, кавычки).

## Запросы к базе данных

**Файл:** [`db-query.js`](../assets/examples/db-query.js)  
Выполнение SELECT запросов к SQLite базе данных. Разрешает только SELECT, проверяет, что файл базы находится в рабочей области, и поддерживает белый список таблиц.

## Генерация конфигурации

**Файл:** [`generate-config.js`](../assets/examples/generate-config.js)  
Генерация конфигурационных файлов из шаблонов. Доступные шаблоны: Express (package.json), Dockerfile, GitHub Actions workflow. Создаёт директории автоматически.

## Использование примеров

1. **Скопируйте** нужный шаблон в `~/.roo/tools/` или `.roo/tools/`
2. **Адаптируйте** параметры под свою задачу
3. **Протестируйте** через `test_tools_interactively.js` или в чате Roo
4. **Обновите** список кастомных инструментов

**Важно:** Все примеры включают базовую валидацию и проверки безопасности. Расширяйте их под свои нужды.

## Выбор правильного шаблона

| Задача | Шаблон | Изменения |
|--------|--------|-----------|
| REST API вызовы | `api-client.js` | Добавить белый список хостов, кастомные заголовки |
| Чтение/запись файлов | `file-ops.js` | Настроить maxSize, кодировку |
| Пакетное переименование | `batch-ops.js` | Изменить валидацию regex паттерна |
| Git операции | `git-ops.js` | Расширить ALLOWED_COMMANDS |
| Уведомления Slack | `slack-notify.js` | Использовать свой вебхук URL |
| Конвертация данных | `transform.js` | Добавить поддержку YAML, TOML |
| SQLite запросы | `db-query.js` | Изменить белый список таблиц |
| Генерация конфигурации | `generate-config.js` | Добавить новые шаблоны (K8s, Terraform) |
